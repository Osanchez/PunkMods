# World & Level Generation

> Part of the PUNK modding docs. Source: decompiled Punk.Main.dll (Unity 6000.3.4f1, Mono).

## Overview

PUNK builds each run's world from a single integer **seed** through a long, deterministic pipeline orchestrated by `LevelGenerator.GenerateLevel`. The level is *not* a tilemap of hand-placed rooms; it is a **graph of circular rooms** (`LevelGraphNode`) connected by **edges/corridors** (`LevelGraphEdge`), which is then "rasterized" into a dense per-cell grid stored in a single `Level` object.

The flow, at a high level:

1. **Seed** is resolved once (`RunDataInstaller`) and installed as a `Seed` service. Every generator seeds its own `Unity.Mathematics.Random` / `Rnd` from it in `Initialize()`.
2. **Graph layout** — required rooms (center + radial) are placed, then random rooms fill the disc, then rooms are connected via Delaunay triangulation (`BowyerWatson`) + a Kruskal minimum spanning tree (`Kruskal`) + a BFS "main path", with dead-ends reconnected.
3. **Biomes** — biome IDs are spread node-to-node across the graph (`BiomeSpreader`), crusts added (`BiomeCrustGenerator`), then baked to a per-cell **biome map** by a Voronoi+Perlin Burst job (`BiomeMapGenerator`), refined by **sub-biomes** (`SubBiomGenerator`).
4. **Dungeons / POIs / stations** are placed onto graph nodes (`DungeonGenerator`, `PoIGenerator`, `StationGenerator`).
5. **Heightmap** — a signed-distance-to-room-surface field is generated by a Burst job (`HeightmapGenerator` → `HeightMapGeneratorJob`); the world **border** and **biome-border mask** are applied.
6. **Rasterization** — each biome's `RasterizationStack` of layers (height range + noise range → cell type) is evaluated per cell by a Burst job (`JRasterizator` → `RasterizationJob`), producing `Level.cellTypes`.
7. **Post-processing** — merged multi-cell tiles, depth map, background cells, outline, plants, entities, scanner areas, fog.

After generation the world is presented in **25×25 segments** (`Level.SegmentSize`) streamed around the camera (`LevelSegmentManager` / `LevelSegmentComponentManager`), with a full-color **map** (`MapDrawer`) and **minimap** (`Minimap`) rendered from the same cell arrays.

Most heavy per-cell work runs in **Burst-compiled Unity Jobs** operating on `NativeArray`s — these are compiled to native code and **cannot be IL-patched by Harmony** (see Modding Notes). The orchestration, config reading, and graph topology code is plain managed C# and *is* patchable.

## Generation Pipeline

Ordered steps inside `LevelGenerator.GenerateLevel(LevelGenerationContext, int seed)`. The public `LevelGenerator.LevelGenerationStep` enum (raised via the static `StepStarted`/`StepFinished` actions) marks the coarse phases shown in **bold**.

1. `context.Initialize(config.levelSize.x, config.levelSize.y)` → `Level.SetSize` allocates all persistent `NativeArray`s (`cellTypes`, `bioms`, `mainBioms`, `heightMap`, etc.), heightMap pre-filled with `1f`.
2. **GenerateGraph** — `GraphGenerator.Generate(context)`:
   - `RequiredRoomsGenerator.PlaceCenterRoom` then `PlaceRadialRooms` (center room + ring of radial main-area rooms from config).
   - `RegularRoomsGenerator.Generate` — random rooms placed inside the disc until `config.roomCount` is reached or `config.maxFailedRoomPlacementAttempts` exhausted, using weighted per-biome `RoomSetup`s.
   - `GraphEdgeGenerator.ConnectRooms` — `BowyerWatson.Triangulate` (Delaunay) → `GenerateMainPath` (BFS) → `Kruskal.MinimumSpanningTree` → `PotentialDungeonLocationCollector.CollectPotentialDungeonLocations` → `ConnectDeadEnds`.
   - `GraphGenerator.SpreadMainAreas` → `BiomeSpreader.SpreadBiomes` (Burst `IJob`) + `BiomeCrustGenerator.Generate` + `CalculateDistancesFromCenter`.
3. `InterestLevelSpreader.SpreadInterestLevel(level)` (Burst `IJob`).
4. `ScannerGenerator.GenerateScanners(level)`.
5. `GraphGenerator.UpdateEdgeSettingsBasedOnBiome(level.graph)` — applies each start-node biome's `edgeSetting` to its edges.
6. **PlaceDungeons** — `DungeonGenerator.PlaceDungeons(context)` (managed; greedy placement onto potential locations).
7. `StationGenerator.GenerateStations(level)`, then `InterestLevelSpreader.SpreadInterestLevel` again.
8. `PoIGenerator.PlacePoIs(context)`.
9. **GenerateBiomeMap** — `BiomeMapGenerator.GenerateBiomeMap` (Burst `BiomeMapGeneratorJob`) writes `level.bioms`; then `SubBiomGenerator.GenerateSubBioms` (copies bioms→mainBioms, then Burst `SubBiomGeneratorJob`).
10. **GenerateHeightMap** — `HeightmapGenerator.GenerateHeightMap` (Burst `HeightMapGeneratorJob`, scheduled per 25×25 segment).
11. `BorderGenerator.GenerateBorder` (Burst `BorderGeneratorJob`) — paints void/border biomes outside the circular play area.
12. `BiomeBorderMaskGenerator.GenerateBorderMask(context, config.biomeBorderWidth)` (Burst job) → `context.biomeBorderMask`.
13. **Rasterization** — `JRasterizator.Rasterize(context, seed)` (Burst `RasterizationJob`) writes `level.cellTypes` using a random world-space `offset` derived from the seed.
14. `MergedCellsGenerator.Generate(level)` — merges adjacent same-type cells into multi-tile `Level.MergedCell` props (uses `UnityEngine.Random`).
15. `DepthMapGenerator.Generate(context)` (Burst `DepthMapGeneratorJob`) → `context.depthMap`.
16. `BackgroundGenerator.Generate(level)` (Burst `BackgroundGeneratorJob`, optional service) → `level.backGroundCellTypes`.
17. `new OutlineGenerator().Generate(context)` — edge-cell detection + traced outline edges (managed).
18. **GrowPlants** — if `config.generatePlants`: `PlantGenerator.GeneratePlants(context, seed)`. Else `level.plants` filled with empty (`plantTypeIndex = 255`).
19. **PlaceEntities** — if `config.generateEntities`: `EntityGenerator.PlaceEntities`, `StationGenerator.InitializeStations`, `RandomObjectGenerator.Generate(context)` (uses `Rnd(seed)` + per-biome `RandomObjectInRooms`).
20. `ScannerAreaGenerator.Generate(level)`.
21. `FogManager.FillInitialFogLevels()` (if present).

Runtime entry point: `GameController.BuildLevel` calls `levelGenerator.GenerateLevel(level, ServiceLocator.Get<Seed>())`, then fires the static `GameController.LevelGenerated` event that the map/lightmap/segment systems subscribe to.

## Classes

### LevelGenerator
- **Kind:** Service (`IGameService`).
- **Implements:** `ILevelGenerator`, `IGameService`.
- **Purpose:** Master orchestrator. Runs the whole pipeline (see Generation Pipeline) by pulling each sub-generator from the `ServiceLocator`.
- **Key fields:** `static bool debugSteps`; `static Action<LevelGenerationStep> StepStarted`; `static Action<LevelGenerationStep,float> StepFinished`; `InterestLevelSpreader interestLevelSpreader`; `currentStep`, `currentStepStartTime`.
- **Key methods:** `UniTask GenerateLevel(Level, int seed)` (wraps a `LevelGenerationContext`); `UniTask GenerateLevel(LevelGenerationContext, int seed)` (the real work); `StartNewStep(step, isFirst)`.
- **Enums:** `LevelGenerationStep { GenerateGraph, GenerateBiomeMap, PlaceDungeons, GenerateHeightMap, Rasterization, GrowPlants, PlaceEntities }`.
- **Relationships:** The single best Harmony anchor for the whole pipeline. All sub-generators are resolved via `ServiceLocator.Get<...>()`.

### DummyLevelGenerator
- **Kind:** Service. **Implements:** `ILevelGenerator`, `IGameService`, `IInitializable`.
- **Purpose:** Debug/fast path — generates a flat single-room world (biome id 6, a 100×100 clear `RectInt`) and places the base. Selected when `LevelGeneratorInstaller.useDummyGenerator` is true.
- **Key methods:** `GenerateLevel(level, seed)`, `PlaceBase(level, position, unlocked, Rnd)`.

### LevelGeneratorInstaller
- **Kind:** MonoBehaviour `IServiceInstaller`.
- **Purpose:** Wires every generation service into the container in `InstallServices`. Picks `LevelGenerator` vs `DummyLevelGenerator` from `useDummyGenerator`, and `levelGeneratorConfig` vs `editorLevelGeneratorConfig` based on `Application.isEditor`.
- **Key fields:** `LevelGeneratorConfig levelGeneratorConfig`, `editorLevelGeneratorConfig`; `BackgroundGenerator backgroundGenerator`; `EdgeColliderBuilder edgeColliderBuilder`; `bool useDummyGenerator`.
- **Modding relevance:** Replacing the installed `ILevelGenerator` here is the cleanest way to swap in a custom generator.

### LevelGeneratorConfig
- **Kind:** ScriptableObject (`CreateAssetMenu "Punk/Level/Level generator config"`).
- **Purpose:** The master tuning asset for world generation. This is the primary data target for most "bigger / more / different" mods.
- **Key fields:** `Vector2Int levelSize`; `Biom defaultBiome, voidBiom, borderBiom`; `bool generateEntities, generateEnemies, generatePlants`; `RequiredRoom centerRoom`; `List<RequiredRoom> radialRooms`; `MinMaxFloat radialRoomsDistanceFromCenter`; `float radialRoomsAngleVariance`; `DungeonWithCount[] dungeons`; `int roomCount`; `int maxFailedRoomPlacementAttempts`; `float margin`; `int biomeBorderWidth`; station distance/clear fields; `LevelGraphEdgeSetting mainPathSetting`; `BiomeMapGenerator.NoiseLayer[] biomeBorderNoiseLayers`; `float borderWidth`; `BiomeMapGenerator.NoiseLayer[] borderInnerNoise`; `PoI stationPoI, scannerPoI`; scanner params; `PoIConfig[] regularPois`.
- **Nested types:** `PoIConfig` (poi, maxAmount, biome filter/blacklist, distances, interest); `struct RequiredRoom` (defautlBiome, mainRoomBiome, poi, roomSetup, biomeSpreadRange, crust fields); `struct DungeonWithCount` (dungeonSetup, `MinMaxInt count`).

### Level
- **Kind:** Service / central data store (`IInitializable`, `IDisposable`).
- **Purpose:** Holds the entire generated world as parallel `NativeArray`s indexed `y * Width + x`, plus the graph and entity manager. All generators read/write these arrays.
- **Key fields:** `LevelGraph graph`; `NativeArray<float> heightMap`; `NativeArray<byte> mainBioms, bioms, cellTypes, foreGroundCellTypes, backGroundCellTypes, luminocity, scannerAreas, fogLevels`; `NativeArray<PlantCell> plants`; `NativeHashMap<int,float> burnLevels`; `NativeArray<HalvedByte> containingMergedCellRelativePosition`; `Dictionary<int,MergedCell> mergedCells`; `NativeHashSet<int2> obstackles`; `EntityManager entityManager`; `static int SegmentSize = 25`; `bool generationFinished`.
- **Key properties:** `Width`, `Height`, `float2 Center`, `Vector2Int SegmentCount`.
- **Key methods:** `SetSize(w,h)` (allocates everything, heightMap=1, mergedCell markers=255); `Initialize()` (grabs registries); cell accessors `GetCellType/SetCell/DestroyCell`; biome `GetBiom/SetBiom/GetMainBiom`; merged cells `PlaceMergedCell/TryGetMergedCellContaining`; foreground/background/luminosity/burn accessors; `GetBaseTransform` (random rotation/mirror via `Unity.Mathematics.Random` seeded by cell hash).
- **Events:** `CellChanged`, `PlantDestroyed`, `ForegroundChanged`, `CellBrokeFromMergedCell` (fire only after `generationFinished`).
- **Nested:** `struct CellChange`, `struct MergedCell` (data + rotation + mirror, with rotated/mirrored `GetTile`).

### LevelGenerationContext
- **Kind:** `IDisposable` scratch container.
- **Purpose:** Holds transient per-generation data not stored on `Level`.
- **Key fields:** `Level level`; `Outline outline`; `NativeArray<bool> edgeCells, biomeBorderMask`; `NativeArray<short> depthMap`; `List<PotentialDungeonLocation> potentialDungeonLocations`; `List<Dungeon> dungeons`; `List<MainArea> mainAreas`; `NativeHashSet<int> mainPathNodes`.
- **Key methods:** `Initialize(width,height)` (calls `level.SetSize` + allocates context arrays), `Dispose()`.
- **Nested:** `struct MainArea { int centerNodeIndex; LevelGeneratorConfig.RequiredRoom requiredRoom; }`.

### Seed / ISeedProvider / SeedText / Rnd / RunDataInstaller
- **`Seed`** — `readonly struct Seed(int value)` with implicit `operator int`; installed as a service so any generator can do `ServiceLocator.Get<Seed>()`.
- **`ISeedProvider`** — `int GetSeed()`.
- **`SeedText`** — MonoBehaviour that displays `Seed: {seed}` + game version.
- **`Rnd`** — wraps `System.Random`; `Range(int/float)`, `Next()`, `Probability(p)`, `RandomDirection()`, `ChangeSeed(seed)`. Used by `JRasterizator`, `RandomObjectGenerator`, `RunData`, and `WeightedDistribution.Distribution`.
- **`RunDataInstaller`** — `GetSeed()` returns `RunArguments.seed`, or `Random.Range(0, int.MaxValue)` when it is `0`; installs `new Seed(...)`. **This is where a fixed seed is decided at runtime.** (`GeneratorTestInstaller` instead uses a `seed` SerializeField, or `DateTime.Now.Millisecond` when `randomSeed`.)

### GraphGenerator
- **Kind:** Service (`IInitializable`, `IDisposable`).
- **Purpose:** Builds the room graph and is the factory for nodes/edges. Seeds `random = new Unity.Mathematics.Random((uint)ServiceLocator.Get<Seed>())` in `Initialize`.
- **Key fields:** `LevelGeneratorConfig config`; `Unity.Mathematics.Random random`; `List<RoomSetup> roomSetups`; `List<NoiseSetting> noises`; `NativeParallelMultiHashMap<byte,NativeRoomSetup> roomSetupsPerBiom`; references to the sub-generators.
- **Key methods:** `UniTask Generate(context)` (orchestrates required/regular rooms + edges + biome spread); `UpdateEdgeSettingsBasedOnBiome(graph)`; `SpreadMainAreas`; `CalculateDistancesFromCenter`; `SetupNoises`; `CreateNode(roomSetup, center, biomeId)`; `SetPoi`, `SetRoomSetup`, `GenerateEdge`; `TryPlaceRandomRoom`; `TryGetRandomRoomSetup(biomId, out)` (weighted by `weightInBiom`).
- **Nested:** `struct NativeRoomSetup`.

### LevelGraph
- **Kind:** `struct LevelGraph(int nodeCount, int edgeCount, int noiseCount) : IDisposable`.
- **Purpose:** Native-collection container for the graph topology.
- **Key fields:** `NativeList<LevelGraphNode> nodes`; `NativeList<LevelGraphEdge> edges`; `NativeList<int2> triangulation, spanningTree`; `NativeList<float> noiseScales`; `NativeParallelMultiHashMap<int,JNoiseLayer> noises`; `int startingNodeIndex`.
- **Key methods:** `AddNode`, `AddEdge`/`RemoveEdge` (maintain `numberOfConnections`), `HasEdge`, `AnyNodeOverlaps`, `RadiusInDirectionFromCenter`, `GetNodeWithCenter`, `SetNodeBiome`, `Dispose`.

### LevelGraphNode / LevelGraphEdge
- **`LevelGraphNode`** (struct): `readonly float2 center`; `float radius, margin, edgeWidth, distanceFromCenter, enemyPowerLevel`; `int noiseIndex, roomSetupIndex, poiId, interestLevel, numberOfConnections`; `byte biomeId`; `Rect bounds`. Methods `UpdateBounds(noiseScale)`, `SetPoi(poi)`.
- **`LevelGraphEdge`** (struct): `int startNodeIndex, endNodeIndex`; `bool isPartOfTheSpanningTree, isMainPath`; `Settings settings` (`width, noiseAmplitude, noiseFrequency, edgeWidth`); `Rect bounds`. Methods `ApplySettings(edgeSetting, ref random)`, `CalculateBounds(graph)`, `GetOtherEnd(thisEnd)`.

### LevelGraphNodeSetting / LevelGraphEdgeSetting
- **`LevelGraphNodeSetting`** (ScriptableObject): `MinMaxFloat noiseAmplitude, noiseFrequency`.
- **`LevelGraphEdgeSetting`** (ScriptableObject): `MinMaxFloat width, noiseAmplitude, noiseFrequency, edgeWidth`; `AnimationCurve widthDistribution`; `GetRandomWidth(ref random)` lerps width by the curve. Referenced by each `Biom.edgeSetting`, `DungeonSetup.mainPathEdge/sidePathEdge`, and `config.mainPathSetting`.

### GraphEdgeGenerator
- **Kind:** Service (`IInitializable`).
- **Purpose:** Connects the placed rooms. `ConnectRooms`: Delaunay triangulate node centers (`BowyerWatson`), generate the BFS **main path** from `startingNodeIndex` to each main-area center (skips every 3rd node), build the **MST** (`Kruskal`, with main-path edges as required edges), collect potential dungeon locations, then `ConnectDeadEnds`.
- **Key methods:** `ConnectRooms(context)`, private `GenerateMainPath`, `ConnectDeadEnds`.

### RequiredRoomsGenerator / RegularRoomsGenerator / RoomSetup
- **`RequiredRoomsGenerator`** (service): `PlaceCenterRoom` (config.centerRoom at level center), `PlaceRadialRooms` (one per `config.radialRooms`, evenly spaced around a random start angle ± `radialRoomsAngleVariance`, at `radialRoomsDistanceFromCenter`). Each becomes a `MainArea`. Seeds its own `Random` from `Seed`.
- **`RegularRoomsGenerator`** (service): loops placing random rooms via `graphGenerator.TryPlaceRandomRoom` until `config.roomCount` nodes or `config.maxFailedRoomPlacementAttempts` failures; rejects rooms outside the `Width/2 - margin` disc.
- **`RoomSetup`** (ScriptableObject): `MinMaxFloat radius, margin, edgeWidth`; `NoiseSetting noiseSetting`; `GameObject gameObjectAtCenter`; `List<PlacedEntity> placedEntities` (`PlacedEntity { float probability; SavableEntity entity; }`).

### InterestLevelSpreader
- **Kind:** Helper with a Burst `IJob` (`SpreadInterestLevelJob`).
- **Purpose:** For each node with a POI, propagates `interestLevel` outward along edges (decrementing per hop). Drives enemy/loot intensity and the debug heatmap.

### BiomeSpreader
- **Kind:** Service with Burst `IJob` (`SpreadBiomesJob`).
- **Purpose:** Flood-fills biome IDs across the graph from spread sources, bounded by `range`, optionally filtered by an existing biome. `SpreadBiomes(level, sources, biomeDepth)` initializes depth to -1 (via `InitNativeArrayWithValuesJob`) then schedules the job.
- **Nested:** `struct BiomeSpreadSource` (constructors from raw ids, from a `PoI`, or from a `MainArea`).

### BiomeCrustGenerator
- **Kind:** Service (`IInitializable`).
- **Purpose:** For main areas flagged `generateCrust`, finds edge-of-biome nodes and spreads a `crustBiome` ring inward, width scaled by `crustWidth` × biome-depth difference. Uses `BiomeSpreader`.

### BiomeMapGenerator
- **Kind:** Helper containing Burst `BiomeMapGeneratorJob` (`IJobParallelFor`).
- **Purpose:** Bakes the graph's per-node biomes into the per-cell `level.bioms`. For each cell it applies a multi-layer Perlin **domain warp** (`config.biomeBorderNoiseLayers`) then assigns the biome of the **nearest node by `distance - radius`** (Voronoi-like).
- **Nested:** `[Serializable] struct NoiseLayer { float noiseFrequency, noiseAmplitude; }` — reused by `BorderGenerator` and `LevelGeneratorConfig`.
- **Burst note:** the per-cell loop is Burst-compiled; not IL-patchable. The managed `GenerateBiomeMap` wrapper that supplies `config` fields *is* patchable.

### Biom
- **Kind:** ScriptableObject (`"Punk/Level/Biom"`, `IIdentifiable<byte>`).
- **Purpose:** Defines a biome's appearance, terrain ruleset, and room/sub-biome content.
- **Key fields:** `byte id`; `bool ignoreWhenParsing`; `Color mapColor, debugColor`; `CellType backgroundCellType`; `AudioClip ambientClip`; `RandomObjectInRooms randomObjectInRooms`; `List<SubBiomSetup> subBioms`; `RasterizationStack rasterizationStack`; `RoomSetupDistribution roomSetups`; `LevelGraphEdgeSetting edgeSetting`; `List<BiomEcosystem> ecosystems`.
- **Nested:** `BiomEcosystem`, `SubBiomSetup { Biom biom; NoiseSetting noise; float threshold; }`, `RoomSetupDistribution(Item)` (weighted).
- **Relationships:** The hub asset — `rasterizationStack` drives terrain cells, `roomSetups` drives room shapes, `subBioms` drives sub-biome overlay, `edgeSetting` drives its corridors.

### BiomMap / SubBiom / SubBiomGenerator / SubBiomGeneratorJob / SubBiomTextureGenerator
- **`BiomMap`** (ScriptableObject): authoring helper that parses a `Texture2D` into `byte[] biomIndexes` by nearest `mapColor`; runtime `GetBiom(position)`.
- **`SubBiom`** (struct): `center, radius, parentBiom, biom, noiseSetting, priority` — used by the editor preview, not the runtime pipeline.
- **`SubBiomGenerator`** (service, Burst job): copies `bioms → mainBioms`, then for each cell, if a noise sample for a parent biome's sub-biome exceeds its `threshold`, overwrites with the sub-biome id.
- **`SubBiomGeneratorJob`** (Burst `IJobParallelFor`): the per-cell evaluation; nested `struct SubBiomData(parentBiomId, biomId, noiseIndex, noiseThreshold)`.
- **`SubBiomTextureGenerator`** (MonoBehaviour): editor visualization of sub-biome noise into a `RawImage`.

### BorderGenerator
- **Kind:** Service (`IInitializable`) with Burst `BorderGeneratorJob`.
- **Purpose:** Outside the circular play area, paints `voidBiom` (beyond radius) and `borderBiom` (within `borderWidth`, with Perlin-warped edge from `config.borderInnerNoise`). Only affects solid cells (`heightMap >= 1`). Radius = `level.Width/2`.

### BiomeBorderMaskGenerator
- **Kind:** Helper with Burst `BiomeBorderGeneratorJob`.
- **Purpose:** Sets `context.biomeBorderMask[i] = true` when a cell is within `borderWidth` (= `config.biomeBorderWidth`) of a different biome along the 4 axes. Consumed by rasterization layers marked `borderOnly`.

### HeightmapGenerator / HeightMapGeneratorJob
- **`HeightmapGenerator`** (service): schedules `HeightMapGeneratorJob` as an `IJobFor` over `Width*Height / (SegmentSize²)` work items.
- **`HeightMapGeneratorJob`** (Burst `IJobFor`): computes the per-cell height as the **minimum** over all overlapping room nodes and corridor edges of an inverse-lerp of distance to the (noise-perturbed) room surface. `0` = deep interior, `1` = solid/outside. Node noise uses `JNoiseLayer.Value` (`noise.cnoise`); edge noise uses `Mathf.PerlinNoise`. This signed-distance field is the main input to rasterization, depth, and background.

### Noise types
- **`NoiseSetting`** (ScriptableObject `"Punk/Level/Noise setting"`): `float scale`; `List<NoiseLayerSetting> noiseLayers`; `Evaluate(x,y)` = scale × weighted average of layers (managed `Mathf.PerlinNoise`). `OnValidate` forces single-layer weight to 1 and marks the first layer `isFirst`.
- **`NoiseLayerSetting`** (`[Serializable]`): `float weight, frequency`; `bool isFirst`; `Value(x,y) = Mathf.PerlinNoise(x*freq, y*freq)`.
- **`JNoiseLayer`** (struct): Burst equivalent; `Value(float2) = (noise.cnoise(pos*frequency)+1)*0.5`. Note the **managed and Burst paths use different noise functions** (Mathf.PerlinNoise vs cnoise) — relevant if you try to reproduce results outside Burst.
- **`NativeNoiseCollection`** (struct, Burst, `IDisposable`): packs a list of `NoiseSetting`s into a `NativeParallelMultiHashMap<int,JNoiseLayer>` with an `offset`; `Evaluate(noiseIndex, pos)`.
- **`Perlin`** (static): a complete classic-Perlin/Fbm implementation with a fixed permutation table. **Not referenced by the main pipeline** (only `PerlinTest`).

### JRasterizator / RasterizationJob / RasterizationStack / RasterizationLayer / JRasterizationLayer
- **`JRasterizator`** (service, `IInitializable`): builds all the native arrays the rasterization job needs — flattens every biome's `rasterizationStack.layers` into `rasterizationLayers`, with per-biome `rasterizationStackStartIndexes`/`Sizes`, plus a shared noise map. Computes a seed-derived world `offset = Rnd(seed).Range(-10000,10000)` for noise sampling, clears all cells, schedules `RasterizationJob`.
- **`RasterizationJob`** (Burst `IJobParallelFor`): for each cell, looks up its biome's layer range and applies each enabled layer in order: if `heightMap` ∈ `[minHeight,maxHeight]` (and, when `useNoise`, a noise sample ∈ `[minNoise,maxNoise]`, and when `borderOnly`, the `biomeBorderMask` is set), writes `layer.cellType`. Later layers overwrite earlier ones.
- **`RasterizationStack`** (ScriptableObject `"Punk/Level/Rasterization stack"`): `List<RasterizationLayer> layers`; managed `Rasterize(cell, offset)` path (used in non-Burst contexts).
- **`RasterizationLayer`** (`[Serializable] struct`): `bool enabled, borderOnly, useNoise`; `CellType cellType`; `MinMaxFloat heightRange, noiseRange`; `NoiseSetting noiseSetting`.
- **`JRasterizationLayer`** (struct): Burst-packed copy (`minHeight/maxHeight/minNoise/maxNoise/noiseIndex/cellType`).

### DepthMapGenerator / OutlineGenerator / BackgroundGenerator / BackgroundGeneratorJob
- **`DepthMapGenerator`** (helper, Burst `DepthMapGeneratorJob`): per cell, empty cells get `-(distance to nearest terrain - 1)`, solid cells get distance to nearest air, both capped at 30 (diamond search). `emptyCellTypes` = cell types flagged `ignoredByDepthMap`. Writes `context.depthMap` (used for shading/decoration depth).
- **`OutlineGenerator`** (helper, managed): marks `context.edgeCells` where a solid cell borders air, then traces connected outline edges with normals into `context.outline` (consumed by colliders/rendering).
- **`BackgroundGenerator`** (MonoBehaviour service, Burst `BackgroundGeneratorJob`): for each empty cell, places the biome's `backgroundCellType` when a Perlin sample (`perlinFrequency`) exceeds a height-lerped `perlinThreshold`. Optional — only runs if installed (`ServiceLocator.TryGet`).

### MergedCellsGenerator / RandomObjectGenerator / RandomObjectInRooms
- **`MergedCellsGenerator`** (service): scans every solid cell and, with probability `cellType.mergedCellProbability`, tries to place a multi-tile `Level.MergedCell` from the cell type's weighted `mergedCells` distribution (trying rotations/mirrors that fit). Uses `UnityEngine.Random` (**not** seeded from `Seed`, so merged-cell layout is **not** reproducible across runs).
- **`RandomObjectGenerator`** (helper): for each graph node, draws a count and a `SavableEntity` from the node-biome's `RandomObjectInRooms` distributions, spawns near the node center. Uses `Rnd(Seed)` but a per-object `UnityEngine.Random.Range` angle.
- **`RandomObjectInRooms`** (ScriptableObject): `IntDistribution itemPerRoomDistribution`, `SavableEntityDistribution entityDistribution`, `List<PoI> poiBlacklist`.

### Dungeon / DungeonGenerator / DungeonSetup / PotentialDungeonLocation(Collector)
- **`DungeonGenerator`** (service, `IInitializable`): `PlaceDungeons` expands `config.dungeons` (each `DungeonWithCount` with a `MinMaxInt count`) into a shuffled list, then greedily places each onto the best `PotentialDungeonLocation` (re-sorting candidates by distance to nearest placed dungeon). Placing a dungeon rewrites graph edges (removes non-spanning edges, applies main/side path edge settings), reassigns node biomes to `placedBiome`, sets end/side room setups, and draws end/side POIs. Seeds `Unity.Mathematics.Random` from `Seed`.
- **`Dungeon`** (class): `dungeonSetup`, `endNodeIndex`, `entranceNodeIndex`, `mainPathLenght`, `HashSet<int> mainPathNodes, allNodes`.
- **`DungeonSetup`** (ScriptableObject `"Punk/Level/Dungeon setup"`): `Biom[] compatibleParentBiomes`; `float minDistanceFromCenter, maxDistanceFromCenter`; `Biom placedBiome`; `MinMaxInt mainPathLength`; `RoomSetup endRoomSetup, sideDeadEndRoomSetup, defaultRoomSetup`; `PoIDistribution endPoi, sidePoi`; `float sidePoiProbability`; `LevelGraphEdgeSetting mainPathEdge, sidePathEdge`; `CanBePlacedInside(biomeId)`.
- **`PotentialDungeonLocation`** (class): `endNodeIndex`, `entranceNodeIndex`, `List<MainNode> mainPath` (each `MainNode` = a node + its side `subGraphs`).
- **`PotentialDungeonLocationCollector`** (helper): walks from each dead-end node along the graph, collecting a linear main path and small side sub-graphs, to produce candidate dungeon sites.

### DungeonGen geometry: BowyerWatson / Kruskal / Triangle / Edge / Point / Intersection
- **`BowyerWatson`** (static): Delaunay triangulation via a super-triangle, returns `HashSet<Triangle>`.
- **`Kruskal`** (static): `MinimumSpanningTree(graph, requiredEdges)` with union-find; required edges (the main path) are pre-merged so they survive.
- **`Triangle`/`Edge`/`Point`** (classes in `DungeonGen`): geometry primitives with circumcircle test, length comparison, ordered vertices, value equality/hash.
- **`Intersection`** (static, `[BurstCompile]`): `Intersect(a1,a2,b1,b2,out intersect)` segment-segment test (used by station/visibility code).

### LightmapGenerator / ParallelLightmapGeneratorJob
- **`LightmapGenerator`** (MonoBehaviour, `IInitializable`, `IDisposable`): on `LevelGenerated` builds a `lightMapTexture` from a Burst job over the whole level; regenerates 25×25 segments when cells change. Bakes `level.luminocity` and `level.foreGroundCellTypes`. Tunables: `falloff`, `gradient`, `backgroundGradient`, `foregroundThreshold`.
- **`ParallelLightmapGeneratorJob`** (Burst `IJobParallelFor`): per-cell light = `falloff - distanceToNearestLightSource` (diamond search); writes luminosity, foreground flag, and a `Color32` (R=solid mask, G=foreground gradient, B=background gradient).

### Level segment streaming: LevelSegmentManager / LevelSegmentComponentManager / LevelSegmentComponent / LevelUtils / LevelTile
- **`LevelSegmentManager`** (MonoBehaviour): computes the camera's current `Vector2Int` segment (floor of camera pos / `SegmentSize`) and fires `CurrentSegmentChanged`.
- **`LevelSegmentComponentManager`** (MonoBehaviour): instantiates/destroys `LevelSegmentComponent` GameObjects within Chebyshev distance ≤ 3 of the current segment (via `level.GetNeighbourSegments`), one per `Update`.
- **`LevelSegmentComponent`** (MonoBehaviour): one 25×25 segment — builds polygon/trigger colliders (`EdgeColliderBuilder`), reacts to cell changes (particles, loot drop via `LootSelector`/`LootFactory`, collider refresh), and dispatches `ICellCollisionListener` callbacks on collision.
- **`LevelUtils`** (static): `GetNeighbourSegments(level, center, range)` extension.
- **`LevelTile`** (class): empty subclass of `UnityEngine.Tilemaps.Tile`.

### LevelChangeBuffer / LevelSnapshot
- **`LevelChangeBuffer`** (MonoBehaviour): accumulates `Level.CellChange`s each frame and re-broadcasts them in one `CellsChanged` event (consumed by segments, lightmap, map drawer).
- **`LevelSnapshot`** (struct): full serialize/deserialize of the world (`mainBioms, bioms, cellTypes, backGroundCellTypes, heightMap, fogLevels, scannerAreas, merged cells, plants`) to/from a `byte[]` via `Buffer.BlockCopy`/`MemoryMarshal`. `Apply(level)` restores it. This is the save/load representation of generated terrain.

### Maps: MapDrawer / Minimap / MapPixel / MapIcon / MapStateIndicator / LevelTextureDisplay
- **`MapDrawer`** (MonoBehaviour): renders the full map texture from cell/biome/plant colors per 25×25 segment, plus a render-texture **fog-of-war**, scanner reveal animation, and per-entity item pixels. `DiscoverWholeMap()` reveals everything. Reads colors from `CellType.colorOnMap`/`mapTexture`, `PlantType.colorOnMap`, `Biom.mapColor`.
- **`Minimap`** (MonoBehaviour, Burst `UpdateTextureJob`): camera-local minimap; per-cell color from `CellType.minimapTexture` (8×8 tiles), with silhouette edges and `voidBiome` → transparent.
- **`MapPixel`** (MonoBehaviour): binds an entity to a single colored pixel on the map (`MapDrawer.AddPixel/ClearPixel`).
- **`MapIcon`** (MonoBehaviour): an icon with an `EntityData Target` + `TargetChanged` event.
- **`MapStateIndicator`** (MonoBehaviour): shows/hides a status text+color on the map.
- **`LevelTextureDisplay`** (MonoBehaviour): **debug/standalone** generation viewer — calls `LevelGenerator.GenerateLevel` directly and renders `Cells`/`Height`/`Edge`/`Depth` textures plus graph/room/POI gizmos. Useful reference for what each array means. Enums `TextureType`, `[Flags] NodeInfo`.

### Grids & distribution: GridHelper / SpatialGrid / Distribution / DistributionItem
- **`GridHelper`** (static): module-grid connectivity helpers (`Move`, `CollectConnectedModulesRecursive`, `IsConnectedWithNeighbour`) — used by ship/module building, not terrain.
- **`SpatialGrid`** (class): buckets `EntityData` by `position / SegmentSize`; `GetEntitiesInSegment`, `Add/Remove`, auto-rebucketing on `entity.Moved`, `EntityMovedToNewBucket` event.
- **`Distribution<T,TItem>`** (abstract, `WeightedDistribution`): weighted random `Draw()` with overloads taking `Rnd` or `ref Unity.Mathematics.Random` and an ignore-list; `Add(value,weight)`. The base for `PoIDistribution`, `SavableEntityDistribution`, `RoomSetupDistribution`, `IntDistribution`, etc.
- **`DistributionItem<T>`** (`[Serializable]`): a `{ float Weight; T Value; }` entry.

## Modding Notes

### Where the seed lives (fixed / custom seed)
- The run seed is chosen in **`RunDataInstaller.GetSeed()`**: it returns `RunArguments.seed`, or `UnityEngine.Random.Range(0, int.MaxValue)` when that is `0`. To force a fixed seed, Harmony-postfix `RunDataInstaller.GetSeed` to return your constant, or patch `RunDataInstaller.InstallServices` to install `new Seed(yourValue)`.
- For test scenes, `GeneratorTestInstaller` already exposes a `seed` field and a `randomSeed` toggle.
- Every generator reads the seed in its `Initialize()` via `ServiceLocator.Get<Seed>()` and constructs `new Unity.Mathematics.Random((uint)(int)seed)` or `new Rnd(seed)`. So overriding the single `Seed` service deterministically changes the whole graph/biome/dungeon layout.
- **Caveats — non-seeded randomness:** `MergedCellsGenerator` and `RandomObjectGenerator`'s spawn angle use `UnityEngine.Random` (global state), so those details are not reproducible from `Seed` alone unless you also call `UnityEngine.Random.InitState`.

### Bigger / smaller levels
- **`LevelGeneratorConfig.levelSize`** (`Vector2Int`) is the single field controlling world dimensions; everything allocates from `level.Width*Height`. Note the border radius is hard-wired to `level.Width / 2` in `BorderGenerator`, and segment count is `Width/25 × Height/25` — keep dimensions multiples of 25 (`Level.SegmentSize`) to avoid partial edge segments.
- More rooms: raise `roomCount` (and `maxFailedRoomPlacementAttempts` so the placement loop doesn't give up). Reduce `margin` to allow rooms nearer the edge.

### Biome control
- Per-biome terrain is entirely data-driven by **`Biom.rasterizationStack`** (a `RasterizationStack` of `RasterizationLayer`s: cellType × heightRange × optional noiseRange × `borderOnly`). Edit/add layers to change what terrain a biome rasterizes to — no code patch needed.
- Biome **placement** on the graph: tune the `RequiredRoom` entries (`defautlBiome`, `mainRoomBiome`, `biomeSpreadRange`, crust fields) and `config.defaultBiome`. Biome **boundary shape** comes from `config.biomeBorderNoiseLayers` (domain warp in `BiomeMapGeneratorJob`) and `biomeBorderWidth`.
- Sub-biomes: add `Biom.SubBiomSetup` entries (`biom`, `noise`, `threshold`).
- New biomes must be registered in **`LevelElementsCollection.allBioms`** (the `IRegistry<Biom,byte>`); ids are bytes, so the practical biome cap is 256.

### More / different rooms & dungeons
- Rooms are weighted per biome via `Biom.roomSetups` (a `RoomSetupDistribution`). Add `RoomSetup` assets and weight them.
- Dungeons: add `DungeonSetup` assets to **`config.dungeons`** (each with a `MinMaxInt count`). `DungeonSetup.compatibleParentBiomes`, `mainPathLength`, and distance-from-center fields gate placement; `endPoi`/`sidePoi` distributions decide rewards.
- POIs: `config.regularPois` (`PoIConfig[]`) plus per-dungeon POI distributions.

### Harmony-patchable vs Burst (NOT patchable)
**Plain managed C# — good Harmony targets** (orchestration, topology, config reading):
- `LevelGenerator.GenerateLevel` (whole pipeline; also the static `StepStarted`/`StepFinished` hooks let you observe phases without patching).
- `GraphGenerator.Generate` / `TryGetRandomRoomSetup` / `CreateNode`, `GraphEdgeGenerator.ConnectRooms` / `GenerateMainPath`, `RequiredRoomsGenerator`, `RegularRoomsGenerator`.
- `DungeonGenerator.PlaceDungeons` / `TryPlace` / `Place`, `PotentialDungeonLocationCollector`.
- `BiomeSpreader.SpreadBiomes` and `BiomeCrustGenerator.Generate` **wrappers** (the inner `SpreadBiomesJob.Execute` is Burst).
- `RandomObjectGenerator.Generate`, `MergedCellsGenerator.Generate`, `OutlineGenerator.Generate`.
- `RunDataInstaller.GetSeed`, `LevelGeneratorInstaller.InstallServices`.
- The geometry classes (`BowyerWatson`, `Kruskal`, `Triangle`, `Edge`, `Point`) are managed.

**Burst-compiled jobs — compiled to native, `[BurstCompile]`, CANNOT be reliably IL-patched by Harmony.** To alter their behavior, patch the **managed scheduler method that fills the job struct** (listed in parentheses) and change the inputs, or disable Burst globally:
- `BiomeMapGeneratorJob` (via `BiomeMapGenerator.GenerateBiomeMap`).
- `SubBiomGeneratorJob` (via `SubBiomGenerator.GenerateSubBioms`).
- `HeightMapGeneratorJob` (via `HeightmapGenerator.GenerateHeightMap`).
- `RasterizationJob` (via `JRasterizator.Rasterize` — change the `RasterizationStack` data instead).
- `BorderGeneratorJob` (via `BorderGenerator.GenerateBorder`).
- `BiomeBorderMaskGenerator.BiomeBorderGeneratorJob`, `DepthMapGenerator.DepthMapGeneratorJob`, `BackgroundGeneratorJob`, `ParallelLightmapGeneratorJob`, `BiomeSpreader.SpreadBiomesJob`, `InterestLevelSpreader.SpreadInterestLevelJob`, `Minimap.UpdateTextureJob`, `Intersection`.

> Tip: a BepInEx config flag to disable Burst (`-burst-disable-compilation` / `BurstCompiler` toggle) makes the above jobs run as managed IL, after which Harmony can patch their `Execute` — at a large performance cost. Otherwise, treat job structs as read-only and influence them only through the managed scheduling code and the ScriptableObject config assets.

### Registries to extend
New cell types, biomes, plant types, etc. are resolved through **`LevelElementsCollection`** (implements `IRegistry<CellType,byte>`, `IRegistry<Biom,byte>`, `IRegistry<PlantType,byte>`, `IRegistry<EntityPlantData,byte>`). Any custom content must be added to its lists and assigned a unique byte id before `Initialize()` builds the lookup dictionaries.
